## 1. 回测的目的

### 1.1 回测不是什么

**回测（Backtesting）**：在**历史数据**上模拟策略的交易过程，计算策略的收益和风险指标。

初学者最常见的误解：

> ❌ "回测结果好 = 策略能赚钱"

正确的理解：

> ✅ **回测的目的是排除差策略，不是证明好策略。**

一个回测结果差的策略，基本可以确定不行。但一个回测结果好的策略，**可能只是过拟合了历史数据**。

类比你学过的 ML：训练集准确率 99% 不代表模型好——你需要测试集上的表现。回测就像在"训练集"上评估模型。

### 1.2 回测结果应该怎么看

不要只看总收益率或夏普比率。关注以下维度：

| 维度 | 要看什么 | 红旗信号（危险信号） |
|------|---------|-----------------|
| 稳定性 | 分年度/分季度表现，不是只看总体 | 某几年特别好、其他年很差 |
| 参数敏感性 | 改一下参数，结果变化大吗？ | 参数改 10%，夏普比率从 2 变成 0.5 |
| 交易次数 | 策略产生了多少次交易？ | 5 年只交易了 20 次——样本太少 |
| 最大回撤 | 最坏情况下亏多少 | MDD > 40%——即使最终赚钱了 |
| 交易成本敏感性 | 加入交易成本后还赚钱吗？ | 加了千分之三的成本就不赚了 |

## 2. 两种回测框架

### 2.1 向量化回测（Vectorized Backtesting）

把整个时间序列当作**向量/矩阵**一次性处理——利用 NumPy/pandas 的向量化运算。

```python
# 向量化回测示意
signals = (prices.rolling(20).mean() > prices.rolling(60).mean()).astype(int)
positions = signals.shift(1)  # 用昨天的信号决定今天的仓位
returns = positions * daily_returns
cumulative = (1 + returns).cumprod()
```

| 优点 | 缺点 |
|------|------|
| 速度极快（利用 NumPy 向量化） | 难以模拟复杂的交易逻辑（如部分成交） |
| 代码简洁 | 难以精确模拟交易成本和滑点 |
| 适合快速原型验证 | 容易不经意间引入前视偏差 |

### 2.2 事件驱动回测（Event-Driven Backtesting）

按时间顺序**逐根 K 线（每根 K 线代表一个时间段的价格信息，如日 K 线包含一天的 OHLCV）**模拟交易过程，像真实交易一样处理每个事件。

```python
# 事件驱动回测示意
for bar in data:
    # 1. 更新市场数据
    portfolio.update_market_data(bar)
    # 2. 策略生成信号
    signal = strategy.generate_signal(bar)
    # 3. 风控检查
    if risk_manager.check(signal):
        # 4. 执行交易（模拟滑点和成本）
        order = execution.submit(signal, slippage=0.001)
        portfolio.update(order)
```

| 优点 | 缺点 |
|------|------|
| 接近真实交易流程 | 速度慢（Python 循环） |
| 精确模拟交易成本和滑点 | 代码复杂 |
| 不容易引入前视偏差 | 开发周期长 |

**实践建议**：先用向量化快速验证想法，有前途的策略再用事件驱动精细回测。

## 3. 回测中的常见偏差

### 3.1 前视偏差（Look-ahead Bias）

**用了回测时点之后才能获得的信息来做决策。**

最隐蔽也最常见的错误。

例子：
- 用**今天的收盘价**计算信号，然后按**今天的收盘价**成交——实际中你不可能在收盘价计算出来之前就以收盘价买入
- 用**全样本**计算均值和标准差来标准化——你不知道未来的均值
- **财报数据**：公司 3 月 31 日的季报可能到 4 月 30 日才公布，如果你在 4 月 1 日就使用了 3 月 31 日的数据，就是前视偏差

如何避免：
- 信号用 $t-1$ 天的数据计算，$t$ 天执行
- 滚动窗口计算统计量，不用全样本
- 财报数据用**实际发布日期**而非报告期末日

### 3.2 幸存者偏差（Survivorship Bias）

**只用现在还存在的股票做回测，忽略了已经退市/倒闭的股票。**

为什么有问题：被退市的股票往往是表现最差的——如果你的数据库里没有这些"失败者"，你的策略会看起来比实际好。

具体数字：某些研究发现，使用包含幸存者偏差的数据回测，策略年化收益率可能被高估 **1–3 个百分点**。

如何避免：使用**包含退市股票**的数据库。中国 A 股的一些数据源（如 Tushare、Wind）提供包含退市股票的完整数据。

### 3.3 选择偏差（Selection Bias）

**测试了很多策略，只展示/记住表现最好的那一个。**

例子：你测试了 100 个策略，其中有 5 个在回测中夏普比率 > 2。你只关注这 5 个。但即使所有 100 个策略都是**随机的**（完全没有预测力），也有可能碰巧有几个表现好的——这是**多重检验（Multiple Testing）**问题。

类比：如果你掷 100 次硬币，连续 5 次正面是小概率事件。但如果你让 1000 个人各掷 100 次，几乎一定有人出现连续 5 次正面——你不能因此说这个人的硬币"有预测力"。

## 4. 过拟合（Overfitting）

### 4.1 策略过拟合 = ML 过拟合

01-overview 已经类比过：策略过拟合和 ML 中的过拟合是**完全相同的问题**。

| ML 中 | 量化回测中 |
|-------|----------|
| 模型参数太多 | 策略参数太多 |
| 拟合训练集噪声 | 拟合历史数据的噪声 |
| 训练准确率高，测试准确率低 | 回测夏普比率高，实盘夏普比率低 |
| 正则化（L1/L2） | 减少参数、简化策略 |

### 4.2 参数越多，过拟合风险越大

一个策略如果有 10 个可调参数（均线周期、阈值、止损点、止盈点……），你几乎**一定**能找到一组参数在历史数据上表现好——但这可能只是"曲线拟合"。

经验法则：**一个策略的自由参数不应该超过 5 个。** 参数越少越好。

### 4.3 过拟合的信号

| 信号 | 含义 |
|------|------|
| 参数稍微改变，结果剧烈变化 | 策略拟合了噪声而非信号 |
| 策略逻辑无法用简单的经济直觉解释 | 可能是数据挖掘的产物 |
| 回测中交易次数太少（< 50 次） | 统计上不显著 |
| 策略只在某个特定时期表现好 | 可能是regime-specific的假象 |

## 5. 防止过拟合的方法

### 5.1 样本外测试（Out-of-Sample, OOS）

最基本的方法：**把数据分成两部分——训练集（in-sample, IS）和测试集（out-of-sample, OOS）**。

```
|---------- IS（用于开发策略和调参）----------|--- OOS（只用一次，验证策略）---|
|         2010 — 2020                        |    2021 — 2025              |
```

**关键规则**：OOS 数据只能用**一次**。如果你看了 OOS 结果后又回去调参，OOS 就变成了 IS——失去了验证意义。

### 5.2 Walk-Forward Analysis（滚动前推分析）

OOS 测试的问题：只有一次机会，样本太少。Walk-Forward 解决了这个问题：

```
Round 1: |--- IS (2010-2015) ---|--- OOS (2016) ---|
Round 2: |--- IS (2011-2016) ---|--- OOS (2017) ---|
Round 3: |--- IS (2012-2017) ---|--- OOS (2018) ---|
Round 4: |--- IS (2013-2018) ---|--- OOS (2019) ---|
```

每一轮用滚动窗口训练，然后在下一年测试。最终把所有 OOS 的结果拼接起来，得到一条**完全样本外**的收益曲线。

优点：
- 产生更多的 OOS 数据点
- 模拟了实际中"定期重新训练"的操作
- 可以看到策略在不同市场环境（牛市、熊市、震荡）中的表现

### 5.3 交叉验证的时间序列版

06 章讲了 Purged K-Fold。这里补充关键要点：**时间序列的交叉验证和 ML 课上学的交叉验证有本质区别——必须保持时间顺序，不能随机打乱。**

### 5.4 参数稳定性检验

不只看"最优参数"，还要看"最优参数附近"的表现：

1. 比如均线策略的快速均线周期是 20 天（最优）
2. 改成 18 天和 22 天，夏普比率变化大吗？
3. 如果变化很小 → 参数稳定 → 好信号
4. 如果变化很大 → 参数不稳定 → 可能是过拟合

画一张**参数敏感性图**：横轴是参数值，纵轴是夏普比率。好的策略应该有一个**宽且平的峰**，而不是尖且窄的尖刺。

## 6. Deflated Sharpe Ratio

### 6.1 问题：多次尝试导致夏普比率膨胀

Bailey 和 López de Prado（2014）提出了一个重要观点：

如果你测试了 $K$ 个策略（或同一策略的 $K$ 组参数），即使所有策略都是**随机的**（真实夏普 = 0），你观察到的最大夏普比率的期望值也不是 0——而是随 $K$ 增长而增大。

这叫**多重检验偏差（Multiple Testing Bias）**——测试得越多，碰巧看起来好的概率越大。

### 6.2 期望最大夏普比率的近似

当从 $K$ 个独立的标准正态分布中取最大值时：

$$E[\max \text{SR}] \approx \sqrt{2 \ln K}$$

具体数字：

| 测试次数 $K$ | 期望最大夏普（假设真实 SR=0） |
|------------|---------------------------|
| 10 | $\approx 2.15$ |
| 100 | $\approx 3.03$ |
| 1000 | $\approx 3.72$ |

**震惊**：即使你的策略完全随机，测试 100 次也能碰巧得到一个夏普比率 3.0 的"好策略"！

### 6.3 Deflated Sharpe Ratio 的思想

**Deflated Sharpe Ratio（DSR）**对观测到的夏普比率做校正，考虑了：
- 测试了多少次（$K$）
- 收益率的偏度（skewness，不对称性）和峰度（kurtosis，厚尾程度）
- 回测时长

校正后的 p 值告诉你：**在考虑了多次测试之后，这个夏普比率是否仍然统计上显著？**

> **可靠度**：Level 2（多重检验问题——统计学共识）；Level 3（DSR 的具体公式和应用——被广泛引用但业界使用程度参差不齐）。

## 7. 回测结果怎么看——总结

### 7.1 检查清单

拿到一个回测结果后，按以下顺序检查：

1. **有没有偏差？** — 前视偏差、幸存者偏差、选择偏差
2. **交易次数够不够？** — 至少 50–100 次交易才有统计意义
3. **参数稳定吗？** — 参数微调后结果不应该剧变
4. **加了交易成本还赚钱吗？** — 没加成本的回测不值一看
5. **样本外表现如何？** — IS 和 OOS 的表现差距不能太大
6. **经济逻辑说得通吗？** — 没有故事的策略比有故事的更可疑
7. **测试了多少次？** — 如果测试了 1000 组参数找到的"最优策略"，夏普比率至少要 3 以上才有意义

> **可靠度**：Level 3（这个检查清单——综合了学术和业界的最佳实践，不同机构的具体标准有差异）。

| 公式 | 含义 | 关键参数 |
|------|------|---------|
